# TimeLine HR — คู่มือเทส Flow

> อัปเดต: 2026-06-01  
> ระบบ: Multi-tenant HR SaaS | เช็คอินผ่าน Line LIFF

---

## Flow Dependencies Map

```
SETUP (Admin ทำครั้งเดียว)
──────────────────────────────────────────
[1] สร้าง Branch  →  ตั้ง GPS + radius + geo_mode  →  Generate QR
[2] สร้าง Shift   →  ตั้งเวลา + threshold + ค่าปรับ
[3] สร้าง Employee  →  assign Branch
[4] ตั้ง Leave Balance  →  กำหนดโควต้าวันลา

EMPLOYEE LINKING (ทำครั้งเดียวต่อคน)
──────────────────────────────────────────
[5] พนักงานเปิด LIFF  →  หน้า Verify  →  ผูก Line account

DAILY FLOW
──────────────────────────────────────────
[6] เช็คอิน    depends on  [1] [2] [3] [5]
[7] เช็คเอาต์  depends on  [6]
[8] ขอลา       depends on  [4] [5]
[9] อนุมัติลา  depends on  [8]
```

---

## Step 1 — ตั้งค่าสาขา (Branch)

**ทำที่:** Admin Web → เมนู **Branch**

| ขั้นตอน | รายละเอียด | ค่าที่แนะนำ |
|---|---|---|
| ชื่อสาขา | ใส่ชื่อสาขา | สาขาทดสอบ |
| ดึง GPS | กดปุ่ม "ดึงตำแหน่งปัจจุบัน" | ต้องอยู่ที่สาขาจริง |
| รัศมี | กำหนดรัศมียอมรับ | `200` เมตร (test ใช้ `500`) |
| Geo Mode | WARN = แจ้งเตือน / BLOCK = บล็อค | `WARN` สำหรับ test |
| บันทึก | กด "เพิ่มสาขา" | |

**ตรวจสอบ:** Card สาขาแสดง badge `⚠️ WARN` หรือ `🚫 BLOCK` + รัศมี

---

## Step 2 — ตั้งค่ากะ (Shift)

**ทำที่:** Admin Web → เมนู **Shift**

| ฟิลด์ | รายละเอียด | ตัวอย่าง |
|---|---|---|
| สาขา | เลือกสาขาจาก Step 1 | สาขาทดสอบ |
| ชื่อกะ | ชื่อกะทำงาน | กะเช้า |
| เวลาเริ่ม | เวลาเข้างาน | `08:00` |
| เวลาเลิก | เวลาเลิกงาน | `17:00` |
| สายระดับ 1 | เริ่มนับว่าสาย | `08:05` |
| ปิดรับเช็คอิน | หลังเวลานี้ = ขาด | `08:30` |
| ค่าปรับ ระดับ 1 | บาท | `50` |
| ค่าปรับ ระดับ 2 | บาท | `200` |

> **สำคัญ:** `ปิดรับเช็คอิน` คือเวลาสุดท้ายที่เช็คอินได้  
> ถ้าสแกนหลังเวลานี้ → ระบบแจ้ง "ไม่มีกะที่เปิดรับ" = นับว่าขาด

**ตรวจสอบ:** Card กะแสดง timeline เวลาถูกต้อง

---

## Step 3 — สร้างพนักงาน (Employee)

**ทำที่:** Admin Web → เมนู **Employee**

| ฟิลด์ | รายละเอียด |
|---|---|
| รหัสพนักงาน | unique ต่อ tenant |
| ชื่อ-สกุล | ชื่อจริง |
| สาขา | **ต้องตรงกับสาขาใน Step 1** |
| วันที่เริ่มงาน | ใช้คำนวณ seniority (วันพักร้อน) |

**ตรวจสอบ:** พนักงานปรากฏในรายการ, ยังไม่มี Line account ผูก

---

## Step 4 — ตั้ง Leave Balance

**ทำที่:** Admin Web → เมนู **Leave Balance**

### วิธีที่ 1: ตั้ง Default แล้ว Apply ทีเดียว

1. กด **"⚙️ โควต้า Default"**
2. ตั้งค่า:

| ประเภท | ค่า default ทั่วไป |
|---|---|
| ลาป่วย | 30 วัน |
| ลากิจ | 3 วัน |
| ลาพักร้อน | 6 วัน |
| ลาชดเชย | 0 วัน |

3. กด **"📋 ใช้กับพนักงานใหม่ที่ยังไม่มีโควต้า"**

### วิธีที่ 2: ตามอายุงาน (Seniority)

1. กด **"⏳ เงื่อนไขอายุงาน"**
2. กำหนด rule เช่น:

| อายุงาน | วันพักร้อน |
|---|---|
| 0–1 ปี | 6 วัน |
| 1–3 ปี | 8 วัน |
| 3–5 ปี | 10 วัน |
| 5+ ปี | 15 วัน |

3. กด **"🔄 คำนวณและนำไปใช้"**

**ตรวจสอบ:** ตารางพนักงานมีตัวเลขโควต้าครบทุกช่อง

---

## Step 5 — ผูก Line Account (Employee Verify)

**ทำที่:** พนักงาน → เปิด LIFF ผ่าน LINE app

> ⚠️ **ต้องทำก่อน** — ถ้าไม่ผูกบัญชี เช็คอิน/ขอลาไม่ได้เลย

```
เปิด LINE app
→ เปิด LIFF URL
→ ระบบดึง LINE Profile อัตโนมัติ
→ แสดงชื่อ + รูป
→ กด "ยืนยัน" → line_user_id ผูกกับ employee_id
→ หน้าหลัก LIFF ขึ้นมา
```

**ตรวจสอบ:** Admin → Employee list → คอลัมน์ LINE status เปลี่ยนเป็น ✅

---

## Step 6 — เช็คอิน (Check-in)

**ทำที่:** พนักงาน → เปิด LIFF → หน้า **เช็คอิน**

```
1. LIFF โหลด → ขอ GPS permission → อนุญาต
2. กดปุ่มกลม 📷
3. LINE เปิดกล้อง → สแกน QR ที่สาขา
4. ระบบทำงาน:
   ✓ ตรวจ GPS vs สาขา (radius)
   ✓ Auto-detect กะจากเวลาปัจจุบัน
   ✓ บันทึก attendance record
5. Popup ขึ้น:
   ✓ เวลาเช็คอิน
   ✓ ชื่อสาขา
   ✓ ชื่อกะ
   ✓ สถานะ (ปกติ / สายระดับ 1 / สายระดับ 2)
   ✓ ค่าปรับ (ถ้ามีและมาสาย)
```

### QR Code — ดึงได้จาก Admin

Admin Web → Branch → กดปุ่ม **QR** → QR ขึ้น → ดาวน์โหลดหรือ copy URL

### Test cases เช็คอิน

| สถานการณ์ | เวลาสแกน | ผลที่คาดหวัง |
|---|---|---|
| มาปกติ | ก่อน `late_threshold_1` | ✅ สถานะ "มาปกติ" |
| สายระดับ 1 | หลัง `08:05` | ⚠️ "สายระดับ 1" + ค่าปรับ |
| สายระดับ 2 | หลัง `08:20` | 🚨 "สายระดับ 2" + ค่าปรับ |
| หลังปิดรับ | หลัง `08:30` | ❌ "ไม่มีกะที่เปิดรับ" |
| นอกพื้นที่ (WARN) | ในเวลา แต่นอก radius | ⚠️ เช็คอินได้ แต่ flag "นอกพื้นที่" |
| นอกพื้นที่ (BLOCK) | ในเวลา แต่นอก radius | ❌ บล็อค เช็คอินไม่ได้ |
| สแกนซ้ำ | หลังเช็คอินแล้ว | ❌ "เช็คอินในกะนี้แล้ว" |

**ตรวจสอบ Admin:** Attendance report → บันทึกขึ้น + badge วิธี (QR) + สถานะถูกต้อง

---

## Step 7 — เช็คเอาต์ (Check-out)

**ทำที่:** พนักงาน → LIFF → หน้า **เช็คเอาต์**

```
1. หน้าโหลด → แสดงกะที่เช็คอินวันนี้ + เวลาเช็คอิน
2. กด "🚪 เช็คเอาต์ตอนนี้"
3. บันทึก check_out_at
4. แสดงระยะเวลาทำงาน (ชม. / นาที)
```

### Test cases เช็คเอาต์

| สถานการณ์ | ผลที่คาดหวัง |
|---|---|
| เช็คเอาต์ปกติ | ✅ แสดงเวลาและ duration |
| เช็คเอาต์ก่อน `min_checkout` | ❌ "ยังเช็คเอาต์ไม่ได้ — รอถึง XX:XX น." |
| ยังไม่ได้เช็คอินวันนี้ | ❌ "ยังไม่ได้เช็คอินวันนี้" |
| เช็คเอาต์ซ้ำ | ✅ แสดงหน้า "เช็คเอาต์แล้ววันนี้" |

---

## Step 8 — ขอลา (Leave Request)

**ทำที่:** พนักงาน → LIFF → หน้า **วันลา** → แท็บ ✏️ ขอลา

```
1. เลือกประเภทการลา (ป่วย / กิจ / พักร้อน / คลอด)
2. เลือกวันที่เริ่ม – สิ้นสุด
   → ระบบนับวันทำงาน (ข้าม Sat/Sun) อัตโนมัติ
3. ใส่เหตุผล
4. กด "📤 ส่งคำขอลา"
5. แสดง "ส่งคำขอแล้ว!"
```

### Test cases ขอลา

| สถานการณ์ | ผลที่คาดหวัง |
|---|---|
| ลาปกติ มีโควต้าพอ | ✅ สถานะ PENDING |
| โควต้าไม่พอ | ❌ "วันลาคงเหลือไม่เพียงพอ" |
| วันทับซ้อนกับที่ขอไว้แล้ว | ❌ "มีวันลาที่ทับซ้อนกันอยู่แล้ว" |

---

## Step 9 — อนุมัติ / ปฏิเสธลา

**ทำที่:** Admin Web → เมนู **Leave**

```
1. เห็นรายการ PENDING
2. กด "อนุมัติ" หรือ "ปฏิเสธ"
   → ถ้าปฏิเสธ ต้องใส่เหตุผล
3. ระบบ:
   → APPROVED: หัก used_days อัตโนมัติ
   → REJECTED: คืนโควต้า
```

**ตรวจสอบ:**
- พนักงาน → LIFF → หน้า Leave → ประวัติ → สถานะเปลี่ยน
- Leave Balance → ยอดเหลือลดลง (กรณี APPROVED)

## Admin Override — กรณีข้อมูลผิด

### ลงเวลาแทน (Manual Check-in)
Admin → Attendance → ปุ่ม **"+ ลงเวลา"** → เลือก กะ / วัน / เวลา

### แก้ไขเวลา
Admin → Attendance → ปุ่ม **"✏ แก้ไข"** → แก้ check_in / check_out

### Reset บันทึก (ลบแล้วเช็คอินใหม่)
Admin → Attendance → ปุ่ม **"🗑"** → ยืนยัน → พนักงานสแกน QR ใหม่ได้

## จุดที่มักพลาด

| อาการ | สาเหตุ | วิธีแก้ |
|---|---|---|
| `EMPLOYEE_NOT_FOUND` | ยังไม่ผูก LINE | พนักงานไปหน้า Verify ก่อน |
| `NO_SHIFT_AVAILABLE` | สแกนนอก window | ตรวจ `late_threshold_2` ยังไม่เกิน |
| `OUTSIDE_GEOFENCE` | อยู่นอก radius (BLOCK) | ลอง radius ใหญ่ขึ้น หรือเปลี่ยนเป็น WARN |
| `INSUFFICIENT_BALANCE` | โควต้าไม่พอ | ไปหน้า Leave Balance ตั้งค่า |
| `TOO_EARLY` | เช็คเอาต์ก่อน min_checkout | รอถึงเวลาที่กำหนด |
| GPS ไม่แม่น | อุปกรณ์/สภาพแวดล้อม | เพิ่ม radius เป็น 300–500m ช่วง test |
| QR สแกนไม่ขึ้น | `scanCode` scope ไม่เปิด | LINE Console → LIFF → เปิด scanCode |
