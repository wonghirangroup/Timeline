# Flow: กะและนโยบายการทำงาน (Shift & Work Policy)
> แนวทางแก้ปัญหา "พนักงานแต่ละคนหยุดต่างกัน โดยไม่ต้องจัดทีละคนทุกเดือน"

---

## ปัญหา (Problem Statement)

```
❌ วิธีเก่า (ระบบทั่วไป)
   HR ต้องมานั่งกำหนด Shift + วันหยุดทีละคน ทุกเดือน
   พนักงาน 50 คน × 12 เดือน = 600 ครั้ง/ปี — เสียเวลามาก

✅ วิธีใหม่ที่ออกแบบไว้
   HR ตั้งค่า Work Policy ให้พนักงานแต่ละคนครั้งเดียว
   ระบบ Auto-apply ตามกฎทุกเดือน
   Admin ทำแค่ "เปิดวันจอง" เดือนละ 1 ครั้งต่อสาขา
```

---

## แนวคิดหลัก — แยก 3 ชั้น

```
┌─────────────────────────────────────────────────────────┐
│  Layer 1: SHIFT (เวลาทำงาน)                             │
│  กำหนดต่อสาขา — ไม่ค่อยเปลี่ยน                          │
│  เช่น กะเช้า 08:00–17:00 / กะบ่าย 14:00–23:00          │
└─────────────────────────────────────────────────────────┘
           ↓ ผูกกับ
┌─────────────────────────────────────────────────────────┐
│  Layer 2: WORK POLICY (นโยบายการทำงาน)                  │
│  กำหนดต่อ "ประเภทพนักงาน" — ตั้งครั้งเดียวอยู่ตลอด      │
│  เช่น พนักงานประจำ / รายวัน / ออฟฟิศ / ผู้บริหาร        │
└─────────────────────────────────────────────────────────┘
           ↓ auto-apply
┌─────────────────────────────────────────────────────────┐
│  Layer 3: MONTHLY SLOT (วันที่เปิดให้จอง)               │
│  Admin เปิดเดือนละ 1 ครั้งต่อสาขา                       │
│  พนักงานจองเองตาม quota ของ Policy ตัวเอง               │
└─────────────────────────────────────────────────────────┘
```

## Work Policy 4 แบบ

### Policy A — พนักงานประจำ (การตลาด / คลัง)

| ข้อกำหนด | ค่า |
|---|---|
| เช็คอิน/เอาต์ | ✅ บังคับ |
| วันหยุดต่อเดือน | **5 วัน** |
| วิธีเลือกวันหยุด | พนักงานจองเองจาก Slot ที่ Admin เปิด |
| หยุดเสาร์-อาทิตย์ | ❌ ไม่อัตโนมัติ (ขึ้นอยู่กับ Slot) |
| หยุดนักขัตฤกษ์ | ✅ ถ้า Admin ตั้งค่า Holiday |
| OT | ✅ คำนวณได้ |
| สาย threshold | 15 นาที |

---

### Policy B — พนักงานรายวัน / Part-time

| ข้อกำหนด | ค่า |
|---|---|
| เช็คอิน/เอาต์ | ✅ บังคับ |
| วันหยุดต่อเดือน | **4 วัน** |
| วิธีเลือกวันหยุด | พนักงานจองเองจาก Slot ที่ Admin เปิด |
| หยุดเสาร์-อาทิตย์ | ❌ ไม่อัตโนมัติ |
| หยุดนักขัตฤกษ์ | ❌ ไม่ได้โดยอัตโนมัติ (ตามสัญญา) |
| OT | ✅ คำนวณได้ (rate ต่างกัน) |
| สาย threshold | 15 นาที |

---

### Policy C — พนักงานออฟฟิศ

| ข้อกำหนด | ค่า |
|---|---|
| เช็คอิน/เอาต์ | ✅ บังคับ (หรือ Optional ตาม Admin) |
| วันหยุดต่อเดือน | **ไม่จำกัด** (หยุดเสาร์-อาทิตย์อัตโนมัติ) |
| วิธีเลือกวันหยุด | **ระบบ Auto-block เสาร์-อาทิตย์** ไม่ต้องจอง |
| หยุดนักขัตฤกษ์ | ✅ Auto-block จาก Holiday Calendar |
| วันลา | ใช้ระบบ Leave Request ปกติ |
| OT | ✅ คำนวณได้ |

---

### Policy D — ผู้บริหาร / Executive

| ข้อกำหนด | ค่า |
|---|---|
| เช็คอิน/เอาต์ | ❌ **ไม่บังคับ** (skip ได้) |
| วันหยุดต่อเดือน | **Flexible** — ไม่มี quota |
| วิธีเลือกวันหยุด | แจ้ง HR ตรงๆ หรือ Leave Request |
| ติดตาม Attendance | ❌ ไม่ต้องติดตาม |
| OT | ❌ ไม่คำนวณ |
| รายงาน | ❌ ไม่ขึ้นในรายงาน Attendance |

## Flow การตั้งค่า (ทำครั้งเดียว)

### HR ตั้งค่า Work Policy ให้พนักงาน

```
HR เพิ่มพนักงาน (Step 9 จาก Onboarding Flow)
  │
  ├─ เลือก Work Policy:
  │   ○ พนักงานประจำ (5 วัน/เดือน)
  │   ○ พนักงานรายวัน (4 วัน/เดือน)
  │   ○ ออฟฟิศ (เสาร์-อาทิตย์)
  │   ○ ผู้บริหาร (Flexible)
  │
  ├─ เลือก Shift (ถ้า Policy ต้องเช็คอิน):
  │   ○ กะเช้า 08:00–17:00
  │   ○ กะบ่าย 14:00–23:00
  │   ○ ไม่มีกะ (ผู้บริหาร)
  │
  └─ บันทึก → ระบบจำ Policy นี้ไว้ตลอด ไม่ต้องตั้งใหม่ทุกเดือน
```

## Flow รายเดือน (Admin ทำแค่ครั้งเดียวต่อเดือน)

### สำหรับ Policy A และ B (ประจำ + รายวัน)

```
ต้นเดือน (Admin/HR ทำ 1 ครั้ง)
  │
  └─ Web App → "ตั้งค่าวันจอง" → เลือก Branch → เลือกเดือน
       → คลิกเลือกวันที่ในปฏิทิน (เช่น 5 วัน)
       → กดบันทึก ✅

จากนั้นพนักงานจัดการเอง (Admin ไม่ต้องทำอะไรอีก)
  │
  ├─ พนักงานประจำ: เห็น 5 วันที่เปิดให้จอง → เลือกได้สูงสุด 5 วัน
  ├─ พนักงานรายวัน: เห็นวันเดียวกัน → เลือกได้สูงสุด 4 วัน
  └─ พนักงานคนไหนไม่จอง = ไม่ได้หยุด (ระบบไม่บังคับ)
```

### สำหรับ Policy C (ออฟฟิศ)

```
ไม่ต้องทำอะไร — ระบบ Auto-block เสาร์-อาทิตย์ให้เอง
  │
  └─ ถ้าพนักงานออฟฟิศพยายามเช็คอินวันเสาร์
       → ระบบแจ้ง "วันนี้เป็นวันหยุดของคุณ"
       → ไม่บันทึก Attendance
```

### สำหรับ Policy D (ผู้บริหาร)

```
ไม่ต้องทำอะไร — ระบบ skip การเช็คอินให้เลย
  │
  └─ ถ้าผู้บริหารเปิด LIFF → ระบบแจ้ง "ไม่ต้องเช็คอิน"
       → ไม่มีหน้าเช็คอิน (redirect ไปหน้า Profile แทน)
```

## Flow การเปลี่ยน Policy (กรณีพิเศษ)

```
พนักงานย้ายแผนก (เช่น รายวัน → ประจำ)
  │
  └─ HR → หน้าพนักงาน → "แก้ไข Work Policy"
       → เปลี่ยนจาก รายวัน → ประจำ
       → ระบบถามว่า "มีผลตั้งแต่เดือนไหน?"
       → บันทึก → Policy ใหม่มีผลทันที
       → ประวัติเก่ายังคงอิง Policy เดิม
```

## กรณี "ขึ้นอยู่กับตัวบุคคล" (Individual Override)

สำหรับพนักงานที่ต้องการข้อยกเว้นเฉพาะคน เช่น
พนักงานประจำคนนึงได้หยุด 6 วัน/เดือน เพราะทำงานมาครบ 5 ปี:

```
Admin/HR → หน้าพนักงาน → "ข้อยกเว้นพิเศษ (Override)"
  │
  ├─ Override เฉพาะเดือนนี้:
  │   "เดือน มิ.ย. 2026 — หยุดพิเศษ 6 วัน"
  │
  └─ Override ถาวร:
      "ตั้งแต่ 1 ม.ค. 2027 — quota เพิ่มเป็น 6 วัน/เดือน"
```

> ไม่ต้องเปลี่ยน Work Policy ของทั้งกลุ่ม — แค่ Override ที่คนเดียว

## สรุป: Admin/HR ต้องทำอะไร?

### ✅ ทำครั้งเดียว (แล้วไม่ต้องแตะอีก)
- สร้าง Work Policy 4 แบบ (ประจำ / รายวัน / ออฟฟิศ / ผู้บริหาร)
- กำหนด Policy ให้พนักงานแต่ละคนตอนเพิ่มพนักงาน

### ✅ ทำเดือนละ 1 ครั้ง (ต่อสาขา ~5 นาที)
- เปิดวันจองหยุดสำหรับ Policy A + B (ประจำ + รายวัน)
  เช่น "มิ.ย. 2026 — สาขาลาดพร้าว: เปิด 5 วัน"
- พนักงานจองเองผ่าน LINE ทั้งหมด ไม่ต้องจัดให้ทีละคน

### ✅ ทำตามเหตุการณ์ (ถ้ามี)
- Override quota เฉพาะคนที่ได้สิทธิ์พิเศษ
- เปลี่ยน Policy เมื่อพนักงานเปลี่ยนสถานะ (เช่น รายวัน → ประจำ)

### ❌ ไม่ต้องทำเลย (ระบบจัดการเอง)
- วันหยุดเสาร์-อาทิตย์ของออฟฟิศ → Auto-block
- การ skip เช็คอินของผู้บริหาร → Auto-skip
- นักขัตฤกษ์ → ใช้ Holiday Calendar ที่ตั้งปีละครั้ง
