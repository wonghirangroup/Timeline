# Flow: โควต้าวันลา & วันพักร้อน (Leave Balance & Vacation Quota)
> ระบบสะสมโควต้าตามอายุงาน + เชื่อมกับการจองวันหยุดประจำเดือน

---

## ความแตกต่างระหว่าง "วันหยุดประจำ" กับ "วันลา"

```
สิ่งที่หลายคนสับสน — จริงๆ มัน 2 เรื่องที่ต่างกัน:

วันหยุดประจำ (Day-off)          วันลา (Leave)
─────────────────────────        ─────────────────────────
สิทธิ์ตามตาราง Work Policy      สิทธิ์จากกฎหมาย/บริษัท
5 วัน/เดือน (ประจำ)             ลาป่วย / ลากิจ / พักร้อน
4 วัน/เดือน (รายวัน)            ใช้เมื่อ "จำเป็น" หรือ "ต้องการพักยาว"
จองผ่าน Monthly Booking         ขอผ่าน Leave Request + รอ Approve
ไม่ตัด Leave Balance            ✅ ตัดจาก Leave Balance
```

---

## ประเภทโควต้าวันลาทั้งหมด

| ประเภท | ได้เมื่อไหร่ | โควต้าต่อปี | ตัด Balance |
|---|---|---|---|
| **ลาป่วย** (SICK) | วันแรกที่เข้างาน | 30 วัน | ✅ |
| **ลากิจ** (PERSONAL) | วันแรกที่เข้างาน | 3 วัน | ✅ |
| **ลาพักร้อน** (VACATION) | ทำงานครบ 1 ปี | ตามอายุงาน | ✅ |
| **ลาคลอด** (MATERNITY) | เมื่อมีเอกสาร | 98 วัน (กฎหมาย) | ✅ |
| **วันหยุดประจำ** (DAY_OFF) | ทุกเดือนตาม Policy | 4–5 วัน/เดือน | ❌ |

---

## โควต้าพักร้อน — เพิ่มตามอายุงาน (Tier System)

```
ทำงาน < 1 ปี      → 0 วัน (ยังไม่ได้สิทธิ์)
ทำงานครบ 1–3 ปี   → 6 วัน/ปี  (กฎหมายขั้นต่ำ)
ทำงานครบ 3–5 ปี   → 8 วัน/ปี
ทำงานครบ 5–10 ปี  → 10 วัน/ปี
ทำงานครบ 10 ปี+   → 15 วัน/ปี
```

> Admin สามารถปรับ Tier นี้ได้ต่อ Company — ด้านบนเป็นค่า default

---

## Flow การได้รับโควต้าพักร้อน (Anniversary Auto-credit)

```
ทุกวัน — ระบบ Cron Job รันตี 00:01 น.
  │
  └─ ตรวจ employees ทุกคนใน tenant ที่ยัง ACTIVE
       │
       ├─ ทำงานครบ 1 ปีพอดีวันนี้ไหม?
       │   (today == start_date + 1 year)
       │   YES →
       │         คำนวณ tier จากอายุงาน
       │         credit VACATION balance ให้พนักงาน
       │         ส่ง LINE notification:
       │           "🎉 คุณได้รับโควต้าพักร้อน 6 วัน
       │            เนื่องจากทำงานครบ 1 ปี ใช้ได้ถึง 31 ธ.ค. 2026"
       │         บันทึกลง leave_balance_logs
       │
       └─ ครบ 3 ปี / 5 ปี / 10 ปีไหม?
           YES → อัปเดต tier → credit ส่วนต่าง (เช่น +2 วัน)
```

### ตัวอย่าง

```
สมชาย เริ่มงาน 1 มิ.ย. 2025
  → 1 มิ.ย. 2026 (ครบ 1 ปี) → ได้ VACATION 6 วัน
  → 1 มิ.ย. 2028 (ครบ 3 ปี) → ได้ VACATION 8 วัน (เพิ่ม 2 วัน)
  → 1 มิ.ย. 2030 (ครบ 5 ปี) → ได้ VACATION 10 วัน (เพิ่ม 2 วัน)
```

## Flow การใช้โควต้า — 2 ช่องทาง

### ช่องทาง A: จองวันหยุดประจำเดือน (Day-off Booking)
> ไม่ตัด Leave Balance — เป็นสิทธิ์ตาม Work Policy

```
พนักงาน LIFF → หน้า "วันลา" → Tab "หยุดสัปดาห์"
  │
  ├─ เห็นวันที่ Admin เปิดให้จอง
  ├─ จองได้ตาม quota ของ Work Policy:
  │   พนักงานประจำ → สูงสุด 5 วัน
  │   พนักงานรายวัน → สูงสุด 4 วัน
  │
  └─ [ถ้ามี Vacation Balance] → เห็นปุ่ม "ใช้วันพักร้อนเพิ่ม"
       → จองวันนอกเหนือจาก quota ปกติได้
       → ระบบตัด VACATION balance อัตโนมัติ
       → ส่ง request รอ Manager approve
```

### ช่องทาง B: ขอลาแบบปกติ (Leave Request)
> ตัด Leave Balance — ใช้เมื่อป่วย / ธุระ / พักยาว

```
พนักงาน LIFF → หน้า "วันลา" → Tab "ขอลา"
  │
  ├─ เลือกประเภท: ลาป่วย / ลากิจ / ลาพักร้อน / ลาคลอด
  ├─ เลือกวันที่ + ระบุเหตุผล
  ├─ ระบบเช็ค balance ว่าเหลือพอไหม
  │   ไม่พอ → แจ้ง "โควต้าไม่เพียงพอ"
  │   พอ → ส่ง request
  │
  └─ Manager Approve → ตัด balance + แจ้ง LINE พนักงาน
```

## กฎการยกโควต้า (Carry-over Policy)

> Admin ตั้งค่าต่อ Company ได้

| กฎ | ตัวอย่าง |
|---|---|
| **ยกได้ทั้งหมด** | เหลือ 4 วัน → ปีหน้าได้ 4+6 = 10 วัน |
| **ยกได้บางส่วน** | ยกได้สูงสุด 3 วัน — เหลือเท่าไหร่ก็ตัดทิ้ง |
| **ยกไม่ได้เลย** | สิ้นปีเหลือเท่าไหร่หมดเลย (Use-it-or-lose-it) |
| **จ่ายเป็นเงิน** | วันที่ไม่ได้ใช้ → คำนวณจ่ายเป็น OT rate |

## Flow สิ้นปี (Year-end Processing)

```
31 ธ.ค. ของทุกปี — ระบบ Cron Job
  │
  ├─ ตรวจ VACATION balance ที่เหลือของพนักงานทุกคน
  │
  ├─ ตาม Carry-over Policy ของ Company:
  │   NONE  → set balance = 0 (หมดทุกส่วน)
  │   LIMIT → ยกได้ max N วัน (ส่วนที่เกินหมด)
  │   ALL   → ยกทั้งหมด
  │   PAYOUT→ บันทึกรอ HR จ่ายเงิน
  │
  ├─ ขึ้น balance ใหม่ปีถัดไป (ถ้า Anniversary ตรงกับต้นปี)
  │
  └─ ส่ง LINE summary ให้พนักงาน:
       "สรุปวันลาปี 2026:
        ใช้ไป: ลาป่วย 2 วัน, พักร้อน 4 วัน
        คงเหลือ: พักร้อน 2 วัน (ยกไปปีหน้า)"
```

## DB Design

```sql
-- โควต้าวันลาของพนักงาน
leave_balances
  id               UUID
  tenant_id        UUID
  employee_id      UUID
  leave_type       ENUM(SICK, PERSONAL, VACATION, MATERNITY)
  year             INT           -- 2026
  total_days       DECIMAL(5,1)  -- โควต้าที่ได้รับในปีนี้
  used_days        DECIMAL(5,1)  -- ใช้ไปแล้ว
  carried_over     DECIMAL(5,1)  -- ยกมาจากปีก่อน
  -- remaining = total_days + carried_over - used_days

-- log การได้รับ/หักโควต้า
leave_balance_logs
  id               UUID
  tenant_id        UUID
  employee_id      UUID
  leave_type       ENUM
  action           ENUM(CREDIT, DEBIT, CARRY_OVER, EXPIRE)
  days             DECIMAL(5,1)
  reason           VARCHAR       -- "ครบ 1 ปี" / "ลาป่วย 10 ม.ค."
  ref_id           UUID          -- อ้างอิง leave_request_id (ถ้ามี)
  created_at       DATETIME

-- config tier พักร้อน (ต่อ tenant ปรับได้)
vacation_tier_configs
  id               UUID
  tenant_id        UUID
  years_from       INT           -- 1
  years_to         INT           -- 3
  days_per_year    INT           -- 6

-- config carry-over (ต่อ tenant)
leave_policies
  tenant_id        UUID
  carry_over_type  ENUM(NONE, LIMIT, ALL, PAYOUT)
  carry_over_max   INT           -- ถ้า LIMIT: ยกได้กี่วัน
  payout_rate      DECIMAL       -- ถ้า PAYOUT: rate เทียบเงินเดือน
```

## ตัวอย่างสถานการณ์จริง

### สถานการณ์ 1: พนักงานประจำ ทำงานครบ 1 ปี อยากพักยาว

```
สมชาย: Work Policy = ประจำ (5 วัน/เดือน)
        ทำงานครบ 1 ปี → ได้ VACATION 6 วัน
        Carry-over: ยกได้สูงสุด 3 วัน

เดือน ก.ค. 2027 — อยากพักยาว 10 วัน (ไปเที่ยวต่างประเทศ)
  ├─ Day-off ปกติ ก.ค.: 5 วัน (ฟรี ไม่ตัด balance)
  └─ ขอ VACATION เพิ่ม 5 วัน (ตัดจาก balance 6 วัน → เหลือ 1 วัน)

รวมหยุดได้: 10 วัน ✅
```

### สถานการณ์ 2: พนักงานใหม่ ยังไม่ครบ 1 ปี ป่วย

```
แนน: เพิ่งเริ่มงาน 3 เดือน
     VACATION balance = 0 (ยังไม่ครบปี)
     SICK balance = 30 วัน

ป่วย 2 วัน → ขอลาป่วย → ตัด SICK 2 วัน (เหลือ 28 วัน) ✅
```

### สถานการณ์ 3: สิ้นปี มีวันพักร้อนเหลือ

```
วัฒน์: VACATION balance เหลือ 4 วัน ณ สิ้นปี
       Company policy: carry_over_max = 3

31 ธ.ค. 2026:
  ├─ ยกไปปีหน้าได้ 3 วัน
  └─ วันที่เกิน (1 วัน) → หมดอายุ / หรือ payout ตาม policy
```

## สรุป: Admin/HR ต้องตั้งค่าอะไรบ้าง?

### ✅ ตั้งครั้งเดียวต่อ Company
- Vacation Tier Config (กี่ปีได้กี่วัน)
- Carry-over Policy (ยกได้/ไม่ได้/จ่ายเงิน)
- โควต้าเริ่มต้นแต่ละประเภทลา (ลาป่วย 30 / ลากิจ 3 ฯลฯ)

### ✅ ระบบทำให้อัตโนมัติ
- Credit VACATION วันครบรอบอายุงานทุกปี
- อัปเดต Tier เมื่อถึง milestone (3/5/10 ปี)
- หักโควต้าเมื่อ Leave Request อนุมัติ
- Reset / Carry-over สิ้นปี
- ส่ง LINE notification ทุก event

### ✅ HR ทำได้เมื่อจำเป็น
- Override โควต้าเฉพาะคน (เช่น โบนัสพักร้อนพิเศษ)
- ดู audit log การได้รับ/หักทุกรายการ
- Export สรุปวันลาทั้งบริษัทเป็น Excel
